# Fondamentaux Product Management

## Purpose
Référentiel des concepts, outils et parcours essentiels pour un PM. Complète le glossaire domaine Galigeo dans CLAUDE.md.

## Glossaire PM

| Terme | Définition |
|-------|-----------|
| **MVP** | Minimum Viable Product — version la plus simple d'un produit permettant de tester avec de vrais utilisateurs |
| **PRD** | Product Requirements Document — document décrivant le quoi, le pourquoi, et pour qui on construit |
| **Roadmap** | Plan de ce qui sera livré, quand, et dans quel ordre de priorité |
| **User Story** | Format : "En tant que [utilisateur], je veux [action] afin de [bénéfice]" |
| **OKR** | Objectives and Key Results — Objective = direction, Key Results = comment mesurer le progrès |
| **Churn** | % d'utilisateurs qui arrêtent d'utiliser le produit sur une période donnée |
| **NPS** | Net Promoter Score — "Recommanderiez-vous ce produit ?" (score de -100 à +100, >50 = excellent) |
| **A/B Testing** | Test de deux versions auprès d'utilisateurs réels. Les données tranchent |
| **Product-Market Fit** | Les utilisateurs restent, convertissent et recommandent sans qu'on ait à pousser |
| **North Star Metric** | Métrique unique qui capture la valeur fondamentale délivrée aux utilisateurs |
| **CES** | Customer Effort Score — mesure la facilité à accomplir une tâche clé dans le produit |
| **JTBD** | Jobs To Be Done — framework centré sur le "travail" que l'utilisateur cherche à accomplir |
| **RICE** | Reach, Impact, Confidence, Effort — framework de priorisation |
| **MoSCoW** | Must have, Should have, Could have, Won't have — priorisation par catégorie |

> **Note** : Le glossaire domaine Galigeo (TM, RetailFocus, Isochrone, etc.) est dans `CLAUDE.md` > Domain Glossary.

## Parcours de carrière PM

| Niveau | Focus principal | Compétences clés |
|--------|----------------|-----------------|
| **Associate PM** | Apprentissage des fondamentaux, exécution, comprendre le cycle de vie produit | Rigueur, curiosité, capacité d'apprentissage |
| **Product Manager** | Ownership d'un périmètre, priorisation, résolution de problèmes bien cadrés | Priorisation, delivery, collaboration cross-fonctionnelle |
| **Senior PM / Team Lead** | Stratégie sur son domaine, réflexion long-terme, collaboration cross-équipes | Vision produit, influence sans autorité, mentorat |
| **Lead / Principal PM** | Vision produit, décisions à fort impact, qualité produit globale | Pensée systémique, arbitrages complexes, leadership technique |
| **Head / Director of Product** | Vision d'entreprise, leadership, construction d'équipes produit solides | Stratégie business, recrutement, culture produit |

### Progression : ce qui change à chaque niveau

- **APM → PM** : Passer de "on me dit quoi faire" à "je sais quoi prioriser"
- **PM → Senior** : Passer de "je livre mon feature" à "je pense stratégie domaine"
- **Senior → Lead** : Passer de "mon périmètre" à "l'ensemble du produit"
- **Lead → Director** : Passer de "le produit" à "l'organisation produit"
